## Step 1 — Query ClickHouse for P1 true/false data up to end of April 2026

This covers the PagerDuty era. P1 = real critical incident; P5 = false P1 (raised for urgency or by mistake).

Call `run_select_query` with the following SQL exactly as written:

```sql
SELECT 
    toStartOfMonth(reported_time) AS month,
    COUNT(DISTINCT IF(reported_priority = 'P1', id, NULL)) AS True_P1_Incidents,
    COUNT(DISTINCT IF(reported_priority = 'P5' AND reported_time >= '2025-06-01', id, NULL)) AS False_P1_Incidents
FROM 
    incidents FINAL
WHERE 
    reported_priority IN ('P1', 'P5')
    AND reported_time >= date_sub(month, 7, toStartOfMonth(now()))
    AND reported_time < '2026-05-01'
GROUP BY 
    month
ORDER BY 
    month;
```

Record each row as `{month, true_p1, false_p1}`. Note: False P1 data only starts from June 2025 as that is when the P5 classification was introduced in PagerDuty.

## Step 2 — Query incident.io for P1 incidents from May 2026 onwards

Call `incident_list` with these parameters:
- `severity: ["01HKQ8WYP01RYH4XD82M9J93KV"]` (P1)
- `created_after: "2026-05-01"`
- `include: ["custom_fields", "timestamps"]`
- `page_size: 50`

Paginate through all pages until no `after` cursor is returned. Collect every incident.

For each incident, read the **P1 validity assessment** custom field (`01KRY7KT75AQJ9KDD9NRF18WJ7`):
- Value `01KRY7KT7591SSZCVWRKCTBM44` → **True P1**
- Value `01KRY7KT75FHMKV0AYHJSVM8JB` → **Not a P1**
- No value set → **Unclassified**

To determine the month, use `reported_at` from the timestamps field (fall back to `created_at` if `reported_at` is absent). Group incidents into calendar months (`YYYY-MM`).

Build a per-month summary:
```
{month: "2026-05", true_p1: N, false_p1: N, unclassified: N, total_p1: N}
```

## Step 3 — Merge both datasets

Combine the ClickHouse rows (Step 1) and the incident.io rows (Step 2) into a single chronological list ordered by month ascending.

For ClickHouse months, set `unclassified: 0` (PagerDuty classified at creation via priority).

For each month compute:
- `false_p1_rate` = `false_p1 / total_p1 * 100` rounded to 0 dp (where `total_p1 = true_p1 + false_p1 + unclassified`)
- Skip division if `total_p1 = 0`

Also compute the following **overall summary** across all months in the window:
- `total_true_p1` = sum of all `true_p1`
- `total_false_p1` = sum of all `false_p1`
- `total_unclassified` = sum of all `unclassified` (incident.io months only)
- `overall_false_rate` = `total_false_p1 / (total_true_p1 + total_false_p1 + total_unclassified) * 100`, rounded to 0 dp

For the trend direction, compare the most recent complete month's `false_p1_rate` against the previous month:
- Improved = ↓ false rate
- Worsened = ↑ false rate
- Unchanged = same rate
